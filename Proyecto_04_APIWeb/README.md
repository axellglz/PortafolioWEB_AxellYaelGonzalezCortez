# Proyecto 4: Consumo de API Web

## Objetivo del Proyecto

Desarrollar una aplicación web capaz de consumir información desde una API externa y mostrar los datos obtenidos de forma dinámica mediante una interfaz web amigable.

## Problema que Resuelve

Permite acceder a información proporcionada por servicios externos a través de APIs, facilitando la integración de datos en aplicaciones web sin necesidad de almacenarlos localmente.

## Tecnologías Utilizadas

* PHP
* HTML5
* CSS3
* JavaScript
* API REST
* cURL
* XAMPP

## Conceptos Aplicados

* Consumo de APIs REST.
* Intercambio de datos en formato JSON.
* Solicitudes HTTP.
* Programación del lado del servidor.
* Integración de servicios web.
* Desarrollo de interfaces dinámicas.

## Descripción del Funcionamiento

La aplicación realiza peticiones a una API externa para obtener información relacionada con diferentes razas de perros. Los datos recibidos son procesados mediante PHP y posteriormente mostrados al usuario en una interfaz web que permite visualizar imágenes e información de manera dinámica.

La comunicación con la API se realiza mediante solicitudes HTTP y los resultados se reciben en formato JSON para su posterior procesamiento.

## Estructura General del Proyecto

* Conexión con API externa.
* Consulta de información.
* Procesamiento de datos JSON.
* Visualización de resultados.
* Interfaz web dinámica.

## Instrucciones de Ejecución

1. Instalar XAMPP.
2. Copiar el proyecto dentro de la carpeta htdocs.
3. Iniciar Apache.
4. Configurar la clave de acceso a la API si es requerida.
5. Acceder al proyecto desde el navegador web.

## Dificultades Encontradas

Durante el desarrollo fue necesario comprender la estructura de las respuestas JSON y establecer correctamente la comunicación con la API para obtener la información deseada.

## Soluciones Implementadas

Se utilizaron funciones de PHP y cURL para realizar solicitudes HTTP, procesar los datos recibidos y mostrarlos de manera organizada dentro de la interfaz web.
